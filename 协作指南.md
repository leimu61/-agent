## 第一次：每个人安装 Hermes + 克隆项目

### 1. 安装 Hermes Agent

```bash
# 用系统终端（PowerShell 或 git-bash）
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

安装完成后，确认能运行：
```bash
hermes --version
```

### 2. 配置 DeepSeek API Key

打开文件 `~/.hermes/.env`（没有就新建），写入：
```
DEEPSEEK_API_KEY=你的key
```

### 3. 克隆项目到本地

```bash
cd ~/Desktop
git clone git@github.com:leimu61/-agent.git ruyi-study-buddy
cd ruyi-study-buddy
```

## 日常工作流（每次开始开发前）

```bash
# 1. 拉取最新代码
cd ~/Desktop/ruyi-study-buddy
git pull

# 2. 编辑你的文件（SKILL.md / tools/xxx.py / docs/xxx.md）

# 3. 提交
git add .
git commit -m "做了什么"

# 4. 推送
git push

# 5. 告诉队友"我更新了，pull 一下"
```

## 在自己的 Hermes 上加载项目 Skill

```bash
# 方法1：启动时加载
hermes -s study-buddy

# 方法2：已启动后在对话中输入
/skill study-buddy

# 方法3：把 SKILL.md 复制到全局 skills 目录
cp ~/Desktop/ruyi-study-buddy/SKILL.md ~/.hermes/skills/study-buddy/SKILL.md
/reload-skills
```

## 常见问题

**Q: 我 push 时报错"non-fast-forward"？**
A: 别人已经推送了新代码，先 `git pull` 再 `git push`。

**Q: pull 时有冲突怎么办？**
A: 大概率不会（各人负责不同文件）。如果冲突了，群里发截图，大家一起看。

**Q: 我改了文件但不想提交？**
A: `git stash` 暂存，做完别的事再 `git stash pop` 恢复。

**Q: 我的 Hermes 怎么看不到别人写的 Skill？**
A: 先 `git pull` 确保文件是最新的，然后 `/reload-skills`。
